## Design patterns

* Builder

#### 2. Structural patterns

* Command

## Design principles
1. Single responsibility
2. Open-closed
3. Loose coupling
4. Use composition rather than inheritance
5. Program to an interface, not an implementation
6. Encapsulate what varies
7. Encapsulation
8. Abstarction
9. Polymorphism
10. Inheritance
